<span id="ai-security"></span>
## 🤖 AI Security

<div class="resource-section ai-section" id="ai-security-section">

<div class="resource-subsection ai-red">
<h3>🔴 AI Red Teaming</h3>
<p class="subsection-desc">Open source tools and frameworks for adversarial testing, attacking, and evaluating AI/ML systems.</p>
<ul class="resource-list">
<li><a href="https://github.com/Azure/PyRIT" target="_blank">PyRIT</a> — Microsoft's Python Risk Identification Toolkit for generative AI. Automates red teaming of LLMs, multimodal models, and AI pipelines.</li>
<li><a href="https://github.com/leondz/garak" target="_blank">Garak</a> — LLM vulnerability scanner. Tests for prompt injection, jailbreaks, data leakage, hallucination, and more across many model providers.</li>
<li><a href="https://github.com/promptsecurity/ps-fuzz" target="_blank">ps-fuzz (Prompt Fuzzer)</a> — Open source tool for fuzzing LLM system prompts to find injection vulnerabilities and weaknesses.</li>
<li><a href="https://github.com/protectai/ai-exploits" target="_blank">AI Exploits (ProtectAI)</a> — Real-world exploits and vulnerabilities found in ML infrastructure and AI tooling. Maintained by the ProtectAI team.</li>
<li><a href="https://github.com/Trusted-AI/adversarial-robustness-toolbox" target="_blank">Adversarial Robustness Toolbox (ART)</a> — IBM's library for defending ML models against adversarial attacks: evasion, poisoning, extraction, and inference attacks.</li>
<li><a href="https://github.com/cleverhans-lab/cleverhans" target="_blank">CleverHans</a> — A Python library to benchmark ML model vulnerability to adversarial examples.</li>
<li><a href="https://github.com/QData/TextAttack" target="_blank">TextAttack</a> — A framework for adversarial attacks, data augmentation, and adversarial training in NLP.</li>
<li><a href="https://github.com/microsoft/promptbench" target="_blank">PromptBench</a> — Microsoft's unified library for evaluating and benchmarking LLM robustness against adversarial prompts.</li>
</ul>
</div>

<div class="resource-subsection ai-blue">
<h3>🔵 AI Blue Teaming & Defense</h3>
<p class="subsection-desc">Open source tools for monitoring, protecting, and securing AI/ML systems and applications in production.</p>
<ul class="resource-list">
<li><a href="https://github.com/protectai/rebuff" target="_blank">Rebuff</a> — Self-hardening prompt injection detector. Uses LLM-based detection, a canary token system, and a shared database of attack signatures.</li>
<li><a href="https://github.com/laiyer-ai/llm-guard" target="_blank">LLM Guard</a> — A comprehensive security toolkit for LLM interactions: input/output scanning for prompt injection, PII, toxicity, and code detection.</li>
<li><a href="https://github.com/NVIDIA/NeMo-Guardrails" target="_blank">NeMo Guardrails</a> — NVIDIA's open source toolkit for adding programmable guardrails to LLM-based conversational applications.</li>
<li><a href="https://github.com/whylabs/langkit" target="_blank">LangKit</a> — An open source text metrics toolkit for monitoring LLM inputs/outputs. Detects prompt injection, toxicity, PII, and sentiment drift.</li>
<li><a href="https://github.com/deadbits/vigil-llm" target="_blank">Vigil</a> — LLM prompt injection and jailbreak detection server with REST API. Scans prompts against known attack signatures and semantic embeddings.</li>
<li><a href="https://github.com/mlflow/mlflow" target="_blank">MLflow</a> — Open source platform for ML lifecycle management including experiment tracking, model registry, and monitoring for drift.</li>
<li><a href="https://github.com/evidentlyai/evidently" target="_blank">Evidently AI</a> — Open source ML and LLM observability framework. Monitors model performance, data drift, and output quality over time.</li>
<li><a href="https://github.com/guardrails-ai/guardrails" target="_blank">Guardrails AI</a> — Open source framework for adding structural, type, and quality constraints to LLM outputs.</li>
</ul>
</div>

<div class="resource-subsection ai-learn">
<h3>📖 AI Security Learning Resources</h3>
<p class="subsection-desc">Open source courses, guides, research, and references for learning AI/ML security concepts.</p>
<ul class="resource-list">
<li><a href="https://github.com/OWASP/www-project-top-10-for-large-language-model-applications" target="_blank">OWASP Top 10 for LLMs</a> — The OWASP project documenting the top 10 most critical vulnerabilities in LLM applications, with mitigations.</li>
<li><a href="https://github.com/dair-ai/Prompt-Engineering-Guide" target="_blank">Prompt Engineering Guide</a> — Comprehensive guide to prompt engineering including adversarial prompting, prompt injection, and jailbreaking techniques.</li>
<li><a href="https://github.com/mitre-atlas/atlas-data" target="_blank">MITRE ATLAS</a> — Adversarial Threat Landscape for Artificial-Intelligence Systems. A knowledge base of adversarial ML tactics and techniques analogous to ATT&amp;CK.</li>
<li><a href="https://github.com/corca-ai/awesome-llm-security" target="_blank">Awesome LLM Security</a> — A curated list of LLM security resources: papers, tools, and real-world vulnerabilities specific to language models.</li>
<li><a href="https://github.com/homakov/awesome-ml-privacy" target="_blank">Awesome ML Privacy</a> — Resources on privacy in machine learning: membership inference, model inversion, differential privacy, and federated learning.</li>
<li><a href="https://github.com/RicardoGeffrard/ai-red-team-exercises" target="_blank">AI Red Team Exercises</a> — Practical exercises and lab scenarios for hands-on adversarial AI testing and red team practice.</li>
</ul>
</div>

<script>
function filterResources(query) {
  query = query.toLowerCase().trim();
  const items = document.querySelectorAll('.resource-list li');
  const subsections = document.querySelectorAll('.resource-subsection');

  items.forEach(function(item) {
    const text = item.textContent.toLowerCase();
    item.style.display = (!query || text.includes(query)) ? '' : 'none';
  });

  subsections.forEach(function(sub) {
    const listItems = sub.querySelectorAll('.resource-list li');
    const visible = Array.prototype.some.call(listItems, function(li) {
      return li.style.display !== 'none';
    });
    sub.style.display = (!query || visible) ? '' : 'none';
  });
}
</script>
